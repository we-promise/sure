## 🇵🇱 Analiza EOD/ROD w kontekście generalized framework

### Czy EOD/ROD pasują do nowego modelu?

| Element | EOD/ROD | Zgodne z generalized framework? |
|---------|---------|----------------------------------|
| subtype | inflation_linked | ✅ Tak |
| rate_type | variable | ✅ Tak |
| coupon_frequency | at_maturity | ✅ Tak (kapitalizacja) |
| inflation_provider | gus_sdp | ✅ Tak |
| product_code | pl_eod, pl_rod | ✅ Tak — label jako kod |

**Polskie obligacje idealnie pasują** — EOD i ROD to inflation_linked z gus_sdp provider.

### Problem: kapitalizacja vs kupony okresowe

Sure Admin pisze żeby dodać obsługę coupon_frequency dla semi_annual, quarterly itp. Ale EOD/ROD mają `at_maturity` — odsetki kapitalizują się do końca, nie ma okresowych kuponów.

**Propozycja rozwiązania:**

```ruby
# bond_lot.rb
def coupon_amount_per_period
  return nil if coupon_frequency == "at_maturity"
  # ... oblicz dla semi_annual, quarterly, monthly, annual
end

# Dla EOD/ROD — zwraca nil (brak kuponów, kapitalizacja)
# Dla TIPS, es_letra z kuponami — oblicza kwotę kuponu
```

### Model decyzyjny:

| product_code | subtype | coupon_frequency | Obsługa |
|-------------|---------|------------------|----------|
| pl_eod | inflation_linked | at_maturity | Kapitalizacja (brak kuponu) |
| pl_rod | inflation_linked | at_maturity | Kapitalizacja (brak kuponu) |
| us_tips_10y | inflation_linked | semi_annual | Obliczany kupon |
| es_letra_3m | zero_coupon | at_maturity | Zero-coupon (brak kuponu) |
| us_t_note_2y | fixed_coupon | semi_annual | Obliczany kupon |

### Dla EOD/ROD — uproszczony model:

Odsetki = (current_cpi_index - purchase_cpi_index) * principal

Nie ma "kuponu" w tradycyjnym sensie — po prostu kapitalizacja. Logika pozostaje prosta:

```ruby
def accrued_interest
  return Money.zero(currency) unless inflation_linked?
  
  current_index = inflation_rate_for(Date.current)
  purchase_index = self.purchase_index || initial_index
  
  (current_index - purchase_index) * amount
end
```

### Rekomendacja Etap A dla EOD/ROD:

1. subtype = inflation_linked
2. product_code = pl_eod / pl_rod
3. inflation_provider = gus_sdp
4. coupon_frequency = at_maturity (pozostaje)
5. coupon_amount_per_period = nil (nie ma kuponu)
6. Logika odsetek: kapitalizacja przez CPI diff

To pozwala zachować obecne EOD/ROD działające bez zmian, a jednocześnie otwiera drzwi dla US/ES bonds z prawdziwymi kuponami.
